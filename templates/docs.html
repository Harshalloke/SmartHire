How SmartHire analyzes your resume vs job descriptions.

How it works

  1. Parsing: Reads PDF/DOCX/TXT and normalizes text (keeps tech tokens like c++, ci/cd, power bi).
  2. Matching: TF-IDF cosine with phrase normalization, alias expansion, and a section-weighted blend (Experience > Skills > Summary).
  3. Insights: Overlap terms, missing keywords, top JD terms, suggested skills (role-aware).
  4. ATS checks: Heuristics for sections, contact info, action verbs, dates, length, readability.
  5. Experience level: Rule-based signals + years extraction.

API

POST /api/analyze — form-data or JSON.

curl -X POST http://127.0.0.1:5000/api/analyze \
  -F "role_hint=Data Analyst" \
  -F "job_description=SQL Excel Python dashboards..." \
  -F "resume=@/path/to/resume.pdf"

Or JSON:

POST /api/analyze
{
  "role_hint": "Data Analyst",
  "job_description": "SQL Excel Python dashboards...",
  "resume_text": "Paste raw resume text here if not uploading a file"
}

Tips for better scores

  • Use clear headings: Experience, Education, Skills.
  • Add years for each role (e.g., 2023–2025).
  • Include action verbs: built, implemented, optimized, delivered.
  • Mirror key JD phrases naturally: “business metrics”, “data cleaning”, “Tableau/Power BI”.
  • Avoid tables/images/symbols that ATS may ignore.
